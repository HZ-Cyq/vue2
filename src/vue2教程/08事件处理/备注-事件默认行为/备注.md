事件的默认行为：
事件有哪些默认行为？

在浏览器语境中，**事件的默认行为（default behavior）**是指：
当某个事件发生时，浏览器在未被显式干预的情况下，自动执行的内置行为。
这些行为由浏览器规范定义，而不是由 JavaScript 或框架（如 Vue）决定。

下面按事件类型系统梳理常见的默认行为。

一、鼠标相关事件的默认行为
1. click

常见默认行为包括：

元素	默认行为
<a href="...">	跳转到指定 URL
<button type="submit">	提交表单
<input type="checkbox">	勾选 / 取消勾选
<input type="radio">	选中单选项
<label for="...">	聚焦或激活对应控件
可编辑区域	获取焦点
<a href="/home" @click.prevent>不会跳转</a>

2. dblclick

选中文本

在部分浏览器中触发缩放（依赖系统设置）

3. mousedown

元素获取焦点

开始文本选择

开始拖拽（可拖拽元素）

4. contextmenu（右键）

打开浏览器右键菜单

<div @contextmenu.prevent>禁用右键菜单</div>

二、键盘事件的默认行为
1. keydown / keypress
按键	默认行为
字母 / 数字	输入字符
Enter（表单中）	提交表单
Tab	切换焦点
Backspace	删除字符 / 返回上一页
Space	滚动页面 / 点击按钮
Arrow keys	移动光标或滚动页面
input.addEventListener('keydown', e => {
  e.preventDefault(); // 阻止字符输入
});

2. keyup

一般没有重要默认行为

多用于监听而非阻止

三、表单相关事件的默认行为
1. submit

向 action 指定地址提交表单

页面刷新或跳转

<form @submit.prevent="onSubmit">

2. reset

重置表单控件为初始值

3. change / input

更新控件值

触发数据同步

（通常不阻止）

四、焦点相关事件的默认行为
1. focus

元素获得输入焦点

光标进入输入框

2. blur

元素失去焦点

触发表单校验（部分浏览器）

五、滚动与视图相关事件
1. wheel / mousewheel

页面或容器滚动

el.addEventListener('wheel', e => {
  e.preventDefault(); // 禁止滚动（需要非 passive）
});

2. touchmove（移动端）

页面滚动

页面缩放（双指）

六、拖拽相关事件
dragstart / dragover / drop

默认行为包括：

拖拽元素

文件拖入浏览器时被打开

放置数据到目标元素

<div @dragover.prevent @drop.prevent></div>

七、资源与导航类事件
1. beforeunload

页面关闭 / 刷新前弹出确认提示

2. load / unload

资源加载完成后的浏览器处理流程

八、哪些事件“几乎没有默认行为”？

这些事件主要用于监听，不涉及阻止：

mousemove

mouseenter / mouseleave

mouseover / mouseout

resize

scroll（本身是结果而非行为）

九、默认行为 vs 冒泡（容易混淆）
机制	控制方法
默认行为	event.preventDefault() / .prevent
事件冒泡	event.stopPropagation() / .stop

两者完全不同，互不替代。
